## Solution convention

Each problem should include:

1. A clear file name (with number or problem slug if possible).
2. The chosen approach (main pattern).
3. Complexity:
	- Time: `O(...)`
	- Space: `O(...)`
4. Covered edge cases.
5. Short comments only when needed.

## Recommended flow for each problem

1. Understand input/output and constraints.
2. Write a brute-force version only if it helps validate the logic.
3. Estimate Big-O.
4. Optimize with a pattern.
5. Test edge cases.
6. Save a clean final version.
